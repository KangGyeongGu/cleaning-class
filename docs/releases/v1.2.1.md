## Highlights

- **GA4·Microsoft Clarity 클라이언트 추적을 재도입**했습니다. 방문자 행동·세션 레코딩·히트맵을 다시 GA4/Clarity 콘솔에서 확인할 수 있습니다.
- 기존 자체 DB 추적과 **dual-write로 병행**해 수치를 교차 검증하는 단계적 전환의 1단계입니다(검증 후 자체 추적 정리 예정).
- 추적 스크립트는 공개 페이지에만 적용되어 `/admin`은 추적에서 제외되며, 견적 폼의 개인정보는 Clarity 레코딩에서 마스킹됩니다.

## Added

- GA4·Microsoft Clarity 클라이언트 추적 재도입 (`analytics`)
  - 기존 `track()` 단일 경로에 GA4 `sendGAEvent` dual-write 추가 — 자체 DB와 GA4에 동시 기록(검증 베이스라인). 호출 컴포넌트 미수정
  - `(public)` 레이아웃에만 GoogleAnalytics·ClarityInit 마운트 → `/admin` 추적 제외
  - 관리자 대시보드 상단에 GA4·Clarity 콘솔 바로가기 링크(서버 env 기반, 미설정 시 숨김)

## Changed

- 견적 폼 이름·연락처·주소 입력에 `data-clarity-mask` 적용 — Clarity 세션 레코딩의 개인정보 캡처 차단 (`analytics`)

## Internal

- 시스템 아키텍처 문서 갱신 — 구성도에 Cloudflare 엣지·Microsoft Clarity 반영, 제거된 GA4 Data API 표기 삭제 (`readme`)
- `@next/third-parties`·`@microsoft/clarity` 의존성 추가
